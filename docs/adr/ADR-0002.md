# ADR-0002: Kubernetes Deployment Strategy

**Status**: Accepted
**Date**: 2026-06-18

## Context
Meridian Stream services must be deployable to Kubernetes with health probes, resource limits, horizontal scaling, and observability wiring.

## Decision
- Stateful services (Redpanda, MinIO) run as StatefulSets with PVCs
- Stateless services (producer, consumer, Prometheus, Grafana) run as Deployments
- All services expose liveness/readiness probes on their metrics port
- Producer scales on CPU; consumer scales on CPU + consumer lag metric
- Config via ConfigMap; secrets via Secret (override with SealedSecrets in prod)
- Deploy via kustomize for environment overlays

## Consequences
- Standard Kubernetes primitives are understood by any operator
- HPA on consumer lag enables demand-driven scaling
- PVCs require a StorageClass; default is cluster-dependent

## Alternatives Considered
- Helm charts: overkill for single-team project; kustomize is simpler
- Raw YAML: no composability; kustomize provides overlay support
